### Problem with Original Reasons
SARD `reason` field is highly generic:
- Safe samples: "This function is a secure function." (45.9% of safe)
- Vulnerable samples: "This function may cause a {CWE_NAME}." (65.7% of vulnerable)
- Unlabeled vuln (cwe_id=None, label=1): "This function first {X} and then {Y}, which may cause a None."
- These have near-zero mutual information with actual code semantics

### UniXcoder Model
- Model: `microsoft/unixcoder-base`
- Encodes both code and text into a unified embedding space
- Cosine similarity between code and description = `sym(code, reason)`
- Lower similarity = more generic/misaligned description
- Higher similarity = more specific/aligned description

## Enhancement Pipeline (Future Work)

### 3-Pass LLM Enhancement Pipeline

For each problematic sample (generic reason):

**Pass 1 — Generate description (Gemma)**:
```
Prompt: You are a vulnerability analysis assistant. Given the following code snippet, generate a concise (1-2 sentence) technical description of the specific vulnerability mechanism. Focus on the concrete unsafe operation. Do NOT use generic phrases like 'may cause a'.

Code:
{code_snippet}
Source: {source}
Sink: {sink}

Generate only the vulnerability description (1-2 sentences):
```

**Pass 2 — Classify CWE ID (Gemma, unlabeled only)**:
```
Given this code snippet and its vulnerability description, identify the exact CWE ID.
Return ONLY the CWE ID number (e.g. "121"). Do not include any other text.

Code:
{code_snippet}

Vulnerability description:
{reason}

CWE ID:
```
- If guessed CWE ∈ valid_89_CWEs: update `cwe_id` field

**Pass 3 — Quality gate (UniXcoder)**:
```
old_sim = sym(code, old_reason)
new_sim = sym(code, new_reason)
If new_sim > old_sim + 0.05: accept
Else: reject, keep old_reason
```
